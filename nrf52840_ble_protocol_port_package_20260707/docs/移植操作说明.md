# nRF52840 主程序集成操作说明

## 1. 工程前提

默认你的工程已经具备：

- nRF52840
- SoftDevice S140
- Nordic UART Service
- NUS RX 可接收电脑端写入
- NUS TX/Notify 可向电脑端发送
- 一个 1 秒周期 timer 或等价调度机制

如果你的工程还没有 NUS，先按 Nordic 官方 NUS peripheral 示例搭好基础 BLE 连接，再接入本协议。

## 2. 加入协议文件

把以下文件加入工程：

```text
include/baby_badge_protocol.h
src/baby_badge_protocol.c
```

工程 include path 增加：

```text
<你的工程路径>/include
```

主程序或 BLE 模块中包含：

```c
#include "baby_badge_protocol.h"
```

## 3. 接收 START / STOP

在 NUS RX 事件中调用：

```c
void on_nus_rx(uint8_t const *data, uint16_t len)
{
    switch (bbp_parse_command(data, len)) {
        case BBP_CMD_START:
            monitoring = true;
            break;

        case BBP_CMD_STOP:
            monitoring = false;
            break;

        default:
            break;
    }
}
```

电脑端点击“开始监测”会写入 `START\n`；点击“停止并导出”会写入 `STOP\n`。

## 4. 每秒发送 sample

准备这些状态变量：

```c
static bool monitoring;
static uint32_t seq;
static uint32_t uptime_ms;
static char frame[BBP_MIN_FRAME_BUFFER_SIZE];
```

1 秒定时器里：

```c
if (!monitoring) {
    return;
}

uptime_ms += 1000;

size_t len = bbp_build_sample_frame(
    frame,
    sizeof(frame),
    ++seq,
    uptime_ms,
    3650,          // 36.50 C，单位 0.01 C
    582,           // 58.2 %RH，单位 0.1 %RH
    "supine",      // 睡姿枚举
    87,            // 电量百分比
    "monitoring",  // 设备状态
    ""             // 没有报警时传空字符串
);

if (len) {
    nus_send_frame(frame, len);
}
```

`nus_send_frame(...)` 是你工程里的 NUS Notify 发送函数。建议内部按 20 字节或当前 MTU 可用长度分片发送，并在 `BLE_GATTS_EVT_HVN_TX_COMPLETE` 后继续发送未完成数据。

## 5. 发送 event

关键事件建议重复发送 3 次。3 次使用不同 `seq`，但保持同一个 `event_id`，电脑端会按 `event_id + type` 去重。

```c
static uint32_t event_id;

void send_kick_event(void)
{
    uint32_t id = ++event_id;

    for (uint8_t i = 0; i < 3; ++i) {
        size_t len = bbp_build_event_frame(
            frame,
            sizeof(frame),
            ++seq,
            id,
            uptime_ms,
            "kick",
            "warning",
            "-2.1C"
        );

        if (len) {
            nus_send_frame(frame, len);
        }
    }
}
```

## 6. NUS 分片发送注意点

不要把一整帧一次性假设能发出去。JSON 帧通常超过 20 字节，必须分片。

推荐策略：

1. 把完整帧放入 TX 队列。
2. 每次最多发送 20 字节，或使用当前连接协商出的 payload 长度。
3. 如果 `ble_nus_data_send` 返回 `NRF_ERROR_RESOURCES` / `NRF_ERROR_BUSY`，保留剩余数据。
4. 收到 `BLE_GATTS_EVT_HVN_TX_COMPLETE` 后继续发送。
5. 不能把一个未发完的帧尾部直接丢弃，否则电脑端 CRC 会失败。

`examples/nrf5_sdk_s140_nus_hooks.c` 里已经给了一个 20 字节分片 + 4 帧队列示例。

## 7. 真实传感器数据替换位置

协议函数入参已经固定，真实传感器只需要映射到这些字段：

```text
temperature_centi  温度，单位 0.01 C，例如 36.50 C -> 3650
humidity_tenth     湿度，单位 0.1 %RH，例如 58.2 %RH -> 582
posture            睡姿枚举字符串
battery_pct        电量百分比，0-100
device_status      normal / idle / monitoring / sensor_error / low_battery
alarm              sample 内报警数组，没报警传 ""
```

睡姿枚举：

```text
supine   正睡
side     侧卧
prone    趴睡
moving   运动中
invalid  数据异常
```

事件类型：

```text
kick          疑似踢被
low_temp      低温预警
prone_alarm   持续趴睡
low_battery   电量过低
sensor_error  传感器异常
```

## 8. 字符串限制

当前 C 端组帧函数没有做通用 JSON 字符串转义。请只传协议枚举、短 ASCII 字符串和简单数值字符串。

不要把带双引号、反斜杠、换行的自由文本直接传给：

```c
bbp_build_sample_frame(... posture/device_status/alarm ...)
bbp_build_event_frame(... type/level/value ...)
```

如果后续必须传自由文本，需要先补 JSON escape。

## 9. 最小联调步骤

1. 烧录带 NUS + 本协议的 nRF52840 固件。
2. 打开电脑端联调包，启动前端和 Python 后端。
3. 页面扫描 BLE 设备。
4. 连接 nRF52840。
5. 点击“开始监测”。
6. nRF52840 收到 `START\n` 后每秒发送 `sample`。
7. 页面实时显示数据和曲线。
8. 触发事件时，nRF52840 连续发送 3 条同 `event_id` 的 `event`。
9. 页面事件列表只显示 1 条去重后的事件。
10. 点击“停止并导出”，nRF52840 收到 `STOP\n` 后停止上报。
