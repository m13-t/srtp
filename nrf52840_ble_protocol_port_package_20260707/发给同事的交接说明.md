# 发给 nRF 主程序同事的交接说明

这是 nRF52840 BLE 协议移植包，用于和电脑端 Python Bleak 后端、网页上位机联调。

如果你只是想先烧板测试，不需要先合入现有主程序，直接打开：

```text
ncs_test_project
```

在 nRF Connect SDK Toolchain 终端中执行：

```bat
west build -p always -b nrf52840dk_nrf52840 .
west flash
```

这个工程不需要真实传感器，烧录后会广播 `BabyBadge-NUS`，收到电脑端 `START\n` 后发送模拟数据。

你真正需要加入固件工程的是：

```text
include/baby_badge_protocol.h
src/baby_badge_protocol.c
```

`examples` 是 nRF5 SDK + S140 + Nordic UART Service 的接入参考，不要直接覆盖主程序。里面演示了：

- NUS RX 收到 `START\n` / `STOP\n` 后切换 monitoring 状态。
- 每秒生成一条 sample。
- 关键事件重复发送 3 次。
- Notify 按 20 字节分片，并在 `BLE_GATTS_EVT_HVN_TX_COMPLETE` 后继续发送。

请先看：

```text
README_先看我.md
docs/移植操作说明.md
docs/协议字段与联调检查清单.md
```

第一阶段先用模拟数据跑通：

```text
nRF52840 -> BLE/NUS -> Python 后端 -> 网页显示
```

网页能看到温湿度、睡姿、电量、事件和曲线后，再把模拟数据替换成 SHT45/JY61P/电池采样的真实数据。替换真实数据时不要改 BLE UUID、JSON 字段名、CRC 规则和 START/STOP 命令。

核心协议：

```text
上行: @<json_payload>*<crc16>\n
下行: START\n / STOP\n
CRC: CRC16-CCITT-FALSE, poly=0x1021, init=0xFFFF
```

CRC 校验向量：

```text
123456789 -> 29B1
```
